# Roadmap Synthesis System - Streamlit UI Specification

## Overview

This specification defines a comprehensive Streamlit-based UI to manage the entire roadmap synthesis system, replacing CLI commands with an intuitive interface.

The UI consolidates all functionality into a cohesive experience:
- Document ingestion and management
- Roadmap synthesis and viewing
- Architecture and competitive assessments
- Open questions and decision management
- Unified knowledge graph exploration
- System configuration and status

---

## Design Principles

### 1. Workflow-Oriented Navigation
Organize around user workflows, not technical features:
- **Build** â†’ Ingest sources, manage documents
- **Synthesize** â†’ Generate and view roadmaps
- **Assess** â†’ Run architecture, competitive, impact assessments
- **Decide** â†’ Manage questions and decisions
- **Explore** â†’ Query the knowledge graph
- **Configure** â†’ System settings and status

### 2. Progressive Disclosure
- Show essential information first
- Details available on demand (expanders, modals)
- Reduce cognitive load

### 3. Context Preservation
- Maintain state across page navigation
- Show relevant context (current roadmap version, pending questions count)
- Breadcrumbs for deep navigation

### 4. Action Feedback
- Clear success/error messages
- Progress indicators for long operations
- Confirmation for destructive actions

def with_loading(message: str):
    """Decorator for operations with loading state."""
    def decorator(func):
        def wrapper(*args, **kwargs):
            with st.spinner(message):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator
